面板的调试（Android / iOS） ​

前置准备

如果您还没有创建或拉取面板工程，请先参考 面板工程的创建和拉取 进行相关操作。

Step 1: 安装依赖 ​

创建好面板工程后，首先需要完成依赖的安装。建议将 npm 源切换到私有仓库，以确保相关依赖能够正常下载：

bash
npm config set registry http://192.168.23.184:4873

配置完成后，在项目根目录下执行以下命令安装依赖：

bash
yarn

Step 2: 运行面板 ​

完成上述所有准备工作后，就可以启动面板项目的本地开发服务了。在项目根目录下运行：

bash
yarn start

启动成功后，进行 App 的配置

Step 3: 获取 PC 的 IP 地址 ​

  1. 按 Win + R 键，打开运行窗口。
  2. 输入 cmd 并回车，打开命令提示符。
  3. 输入以下命令并回车：
cmd
ipconfig

在输出结果中找到类似以下内容，提取 IPv4 地址：

text
Ethernet adapter Ethernet:

   IPv4 Address. . . . . . . . . . . : 10.55.80.11

Step 4: 安装移联万物 App（Debug 版本） ​

在进行真机调试前，您需要安装“移联万物”的 Debug 版本 App。您可以扫描下方对应的二维码进行下载和安装（移联万物 Debug v3.1.2）：

安装完成后，请打开并注册/登录 App。 如果您的账号下还没有绑定任何设备，可以前往 QuecCloud 物联网开发者平台 创建一台虚拟设备，然后在 App 端完成绑定。

Step 5: 配置面板调试 IP 地址 ​

获取到 IP 地址并登录 App 后，需要进行 App 侧的 IP 地址配置以连接本地开发服务：

  1. 点击屏幕上的 DoKit 悬浮图标：

  2. 在弹出的面板中，点击 面板调试：

  3. 开启 面板调试开关，并在下方选择 本地面板 debug 调试：

  4. 返回到 App 首页并点击刚才绑定好的设备，此时由于尚未配置归属，会出现报错页面：

  5. 摇动手机，此时屏幕下方会弹出开发菜单选项，选择 Change Bundle Location：

  6. 输入在 Step 3 中获取的 PC IP 地址，并加上端口号 8081，例如：

    text
    10.55.80.11:8081

  7. 点击 APPLY CHANGES：

配置完成后，即可进入并实时预览您在本地运行的面板工程。
