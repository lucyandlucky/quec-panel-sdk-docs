物模型的初始化 ​

在使用物模型之气，必须在全局进行 Store 的初始化，通常我们在 App 或全局 Context 的入口处进行。

createDeviceModelStore<T>(params?) ​

初始化全局物模型 Store（单例模式，多次调用返回同一实例）。通过泛型传入物模型类型，可以让 IDE 获得完善的类型提示。

参数类型说明
modelClassListArray<{ classFilterMethod, listName }>可选，将物模型按 code 规则自动归类
translateCode(code: string) => string可选，code 转换函数，默认 camelCase
avoidReadingTSLIdnumber[]可选，初始化时跳过读取的物模型 ID 列表

示例：

ts
import { createDeviceModelStore } from '@quec/panel-model-kit'

export const store = createDeviceModelStore<MyModelType>()

createUseModel<T>(store) ​

基于已创建的 Store 生成 useModel Hook，返回应用使用的物模型状态集合。

示例：

ts
import { createUseModel } from '@quec/panel-model-kit'

export const useModel = createUseModel<MyModelType>(store)

// 在组件中使用
// const models = useModel()

useModelInit() ​

返回初始化函数，调用后从云端拉取物模型，并自动向设备发起一次查询以获取当前状态。

示例：

tsx
import { useEffect } from 'react'
import { useModelInit } from '@quec/panel-model-kit'

const AppContainer = ({ device }) => {
  const initModel = useModelInit()

  useEffect(() => {
    if (device) {
      initModel(device.productKey, device.deviceKey)
    }
  }, [initModel, device])

  return null;
}

getDeviceModelStore() ​

获取 Store 实例，可用于直接订阅细粒度状态变化（例如在非 React 环境中使用，或为了极致性能订阅特定字段）。该方法必须在 createDeviceModelStore 调用后使用。

modelKitParamsManager ​

全局单例，用于注册面板级别的通用 UI 回调函数，例如加载动画、操作提示等。

ts
modelKitParamsManager.registerCallbacks({
  loadingCallback: () => { /* 显示 loading */ },
  loadingDismissCallback: () => { /* 隐藏 loading */ },
  tipCallback: (msg) => { /* Toast 提示 */ },
})
