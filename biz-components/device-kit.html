DeviceKit ​

@quec/panel-device-kit 是一个面向移联万物（QuecCloud）物联网平台的设备信息状态管理工具库。它基于 Zustand 状态管理，提供设备信息存储、在线状态监听、设备列表分页管理等完整能力，用于在 React Native 面板中高效管理设备和设备列表。

安装 ​

bash
npm install @quec/panel-device-kit
bash
yarn add @quec/panel-device-kit

注意

本库依赖以下 peerDependencies，请确保它们已在宿主项目中安装：@quec/rn-businesspanel-module、@quec/rn-device-module、@quec/rn-language-module、@quec/rn-log-module、i18n-js、zustand、react、react-native。

快速开始 ​

1. 初始化 Store 并处理传入参数 ​

在应用最顶层（如 App.tsx）初始化设备 Store，并处理 App 传入的 props：

tsx
// App.tsx
import { createDeviceStore, createDeviceListStore, useSetProps } from '@quec/panel-device-kit'

// 初始化 Store（仅需调用一次）
createDeviceStore({
  offlineTip: () => console.log('设备离线'),
  onlineTip: () => console.log('设备上线'),
})
createDeviceListStore()

const App = (props) => {
  const setProps = useSetProps()

  useEffect(() => {
    const propsProcessed = setProps(props)
    global.props = propsProcessed
  }, [setProps])

  return <AppContainer />
}

2. 注册事件监听 ​

在入口容器组件中注册设备状态、设备列表等事件监听：

tsx
// AppContainer.tsx
import {
  useInitUpdateDeviceStateEvent,
  useInitDeviceInfoUpdateEvent,
  useInitDeviceListUpdateEvent,
  useDeviceListRefresh,
  deviceKitParamsManager,
} from '@quec/panel-device-kit'

const AppContainer = () => {
  const refreshDeviceList = useDeviceListRefresh()

  // 设备在线/连接状态监听
  useInitUpdateDeviceStateEvent(
    (event) => { /* 在离线状态更新回调 */ },
    (event) => { /* 连接状态更新回调 */ }
  )

  // 设备信息更新监听（如切换设备）
  useInitDeviceInfoUpdateEvent(() => {
    refreshDeviceList()
  })

  // 设备列表更新监听
  useInitDeviceListUpdateEvent({
    enableAutoCheckoutDevice: true,
    refreshDeviceListEventCallback: null,
    refreshDeviceListSuccessCallback: null,
  })

  // 注册全局 UI 回调
  deviceKitParamsManager.registerCallbacks({
    loadingCallback: global.loading,
    loadingDismissCallback: global.loadingDismiss,
    tipCallback: global.toast,
  })

  return <>{/* ... */}</>
}

3. 在业务页面中使用 ​

tsx
// AnyPage.tsx
import { useDevice, useDeviceOnline } from '@quec/panel-device-kit'

const AnyPage = () => {
  const device = useDevice()
  const isOnline = useDeviceOnline()

  return (
    <>
      <Text>设备名称：{device?.deviceName}</Text>
      <Text>状态：{isOnline ? '在线' : '离线'}</Text>
    </>
  )
}

API 参考 ​

设备状态管理 ​

初始化 ​

createDeviceStore(params) ​

初始化设备状态 Store（单例模式）。

参数类型说明
offlineTipFunction可选，设备离线时的提示回调
onlineTipFunction可选，设备上线时的提示回调

查询 Hooks ​

Hook返回值说明
useDevice()DeviceModel | null获取当前设备信息
useDeviceProps()PropsModel | null获取 App 传入的初始参数
useDeviceOnline()boolean获取设备在线状态
useDeviceState()number获取设备状态码（位掩码）
useDeviceFamily()FamilyModel | null获取家庭信息
useDeviceArea()string | null获取区域信息
useDeviceProductUrl()string | null获取产品说明书链接
useDeviceIsOnlyBle()boolean设备是否为纯蓝牙设备
useDeviceIsConnecting()boolean设备是否正在连接中
useDeviceWsOnline()booleanWebSocket 通道是否在线
useDeviceLanOnline()boolean局域网通道是否在线
useDeviceBleOnline()boolean蓝牙通道是否在线
useEndUserId()string | undefined获取终端用户 ID
useUserAccount()string | undefined获取当前用户账号
useDeviceStoreState()States & Actions获取所有状态（慎用，易引起频繁刷新）

操作 Hooks ​

Hook返回值说明
useSetProps()(props) => PropsModel设置初始参数并同步更新设备
useUpdateDevice()(device) => void更新设备信息
useUpdateDeviceName()(name) => void更新设备名称
useUpdateOnlineStatus()(state) => void更新在线状态
useGetDeviceStatus()() => Promise<number>主动拉取设备状态
useCheckCurrentDeviceExist()() => Promise<boolean>检查当前设备是否还存在

扩展 ​

  • getDeviceStore()：获取 Store 实例（需先调用 createDeviceStore 初始化）。

设备列表状态管理 ​

初始化 ​

createDeviceListStore(params?) ​

初始化设备列表 Store（单例模式）。

参数类型默认值说明
pageSizenumber15每页设备数量
isSubScribeAllDevicebooleanfalse是否订阅所有设备
processDeviceList(device) => T—自定义设备列表数据处理函数
subscribeCodeListstring[]—订阅设备物模型 code 列表

查询 Hooks ​

Hook返回值说明
useDeviceList()DeviceModel[]获取设备列表
useDeviceListStoreState()States & Actions获取设备列表所有状态

操作 Hooks ​

Hook返回值说明
useDeviceListRefresh()(success?, fail?) => void刷新设备列表（重置为第一页）
useDeviceListLoadMore()(success?, fail?) => void加载下一页设备

扩展 ​

  • getDeviceListStore()：获取 Store 实例（需先调用 createDeviceListStore 初始化）。

事件监听 ​

useInitUpdateDeviceStateEvent(stateCallback?, connectingCallback?) ​

监听设备在线/连接状态变化，通常在入口组件中调用一次。

参数说明
stateCallback在离线状态更新时触发
connectingCallback连接状态更新时触发

useInitDeviceInfoUpdateEvent(callback?) ​

监听设备信息更新事件（如 App 通知设备已切换/更新）。

useInitDeviceListUpdateEvent(params?) ​

监听设备列表更新事件。

参数类型说明
enableAutoCheckoutDeviceboolean是否自动切换设备（当前设备被删除时切换到列表第一个）
refreshDeviceListEventCallback(event) => void列表更新事件回调
refreshDeviceListSuccessCallback(list) => void列表刷新成功回调

注意

以上事件监听 Hook 应在入口组件中仅调用一次，避免重复注册。

监听事件名称 ​

事件常量值说明
EVENT_TYPE_DEVICE_ONLINEonDeviceOnlineState设备在离线状态更新
EVENT_TYPE_DEVICE_CONNECTINGonDeviceConnectingState设备连接状态更新
EVENT_TYPE_REFRESH_DEVICE_LISTQuecRefreshDeviceList设备列表更新
EVENT_TYPE_REFRESH_DEVICEQuecRefreshDeviceInfo设备信息更新（更换设备）

设备列表工具方法 ​

changeDevicePanel(newDevice) ​

向 App 发起面板切换请求。如果新设备归属同一面板，App 只会发出 QuecRefreshDeviceInfo 事件通知；否则会重新加载新面板。

checkoutEmptyPanel() ​

当设备列表为空时，主动切换到托底面板。

回调管理器 ​

deviceKitParamsManager ​

全局单例，用于注册面板级别的 UI 回调。

ts
deviceKitParamsManager.registerCallbacks({
  loadingCallback: () => { /* 显示 loading */ },
  loadingDismissCallback: () => { /* 隐藏 loading */ },
  tipCallback: (msg) => { /* Toast 提示 */ },
})

设备在线状态说明 ​

设备在线状态通过位掩码表示，支持以下通道组合：

状态枚举值说明
ALL_OFFLINE0全部离线
WS_ONLINE1WebSocket 在线
WIFI_ONLINE2Wi-Fi 在线
WS_WIFI_ONLINE3WebSocket + Wi-Fi
BLE_ONLINE4蓝牙在线
WS_BLE_ONLINE5WebSocket + 蓝牙
WIFI_BLE_ONLINE6Wi-Fi + 蓝牙
ALL_ONLINE7全部在线

可通过 useDeviceWsOnline()、useDeviceLanOnline()、useDeviceBleOnline() 分别获取各通道状态。

类型定义 ​

DeviceModel ​

设备信息类型，主要字段：

字段类型说明
deviceKeystring设备唯一标识
productKeystring产品标识
deviceNamestring设备名称
onlineStatusnumber在线状态（0 离线 / 1 在线）
accessTypestring访问类型（0 直连 / 1 网关 / 2 子设备）
networkTypestring网络类型（1 WiFi / 2 蜂窝 / 3 NB / 5 蓝牙）
deviceTypenumber设备类型（1 自有 / 2 分享）
capabilitiesBitmasknumber能力位掩码

字段类型说明

familyModeEnabledboolean家居模式是否开启
familyRolestring家庭角色
endUserIdstring终端用户 ID
userAccountstring当前用户账号

FamilyModel ​

家庭信息类型：

字段类型说明
