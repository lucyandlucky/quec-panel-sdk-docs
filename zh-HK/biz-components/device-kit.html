DeviceKit ​

@quec/panel-device-kit 是一個面向移聯萬物（QuecCloud）物聯網平台的裝置資訊狀態管理工具庫。它基於 Zustand 狀態管理，提供裝置資訊儲存、線上狀態監聽、裝置列表分頁管理等完整能力，用於在 React Native 面板中高效管理裝置和裝置列表。

安裝 ​

bash
npm install @quec/panel-device-kit
bash
yarn add @quec/panel-device-kit

注意

本庫依賴以下 peerDependencies，請確保它們已在宿主專案中安裝：@quec/rn-businesspanel-module、@quec/rn-device-module、@quec/rn-language-module、@quec/rn-log-module、i18n-js、zustand、react、react-native。

快速開始 ​

1. 初始化 Store 並處理傳入參數 ​

在應用最頂層（如 App.tsx）初始化裝置 Store，並處理 App 傳入的 props：

tsx
// App.tsx
import { createDeviceStore, createDeviceListStore, useSetProps } from '@quec/panel-device-kit'

// 初始化 Store（僅需呼叫一次）
createDeviceStore({
  offlineTip: () => console.log('裝置離線'),
  onlineTip: () => console.log('裝置上線'),
})
createDeviceListStore()

const App = (props) => {
  const setProps = useSetProps()

  useEffect(() => {
    const propsProcessed = setProps(props)
    global.props = propsProcessed
  }, [setProps])

  return <AppContainer />
}

2. 註冊事件監聽 ​

在入口容器元件中註冊裝置狀態、裝置列表等事件監聽：

tsx
// AppContainer.tsx
import {
  useInitUpdateDeviceStateEvent,
  useInitDeviceInfoUpdateEvent,
  useInitDeviceListUpdateEvent,
  useDeviceListRefresh,
  deviceKitParamsManager,
} from '@quec/panel-device-kit'

const AppContainer = () => {
  const refreshDeviceList = useDeviceListRefresh()

  // 裝置線上/連線狀態監聽
  useInitUpdateDeviceStateEvent(
    (event) => { /* 在離線狀態更新回撥 */ },
    (event) => { /* 連線狀態更新回撥 */ }
  )

  // 裝置資訊更新監聽（如切換裝置）
  useInitDeviceInfoUpdateEvent(() => {
    refreshDeviceList()
  })

  // 裝置列表更新監聽
  useInitDeviceListUpdateEvent({
    enableAutoCheckoutDevice: true,
    refreshDeviceListEventCallback: null,
    refreshDeviceListSuccessCallback: null,
  })

  // 註冊全域 UI 回撥
  deviceKitParamsManager.registerCallbacks({
    loadingCallback: global.loading,
    loadingDismissCallback: global.loadingDismiss,
    tipCallback: global.toast,
  })

  return <>{/* ... */}</>
}

3. 在業務頁面中使用 ​

tsx
// AnyPage.tsx
import { useDevice, useDeviceOnline } from '@quec/panel-device-kit'

const AnyPage = () => {
  const device = useDevice()
  const isOnline = useDeviceOnline()

  return (
    <>
      <Text>裝置名稱：{device?.deviceName}</Text>
      <Text>狀態：{isOnline ? '線上' : '離線'}</Text>
    </>
  )
}

API 參考 ​

裝置狀態管理 ​

初始化 ​

createDeviceStore(params) ​

初始化裝置狀態 Store（單例模式）。

參數類型說明
offlineTipFunction可選，裝置離線時的提示回撥
onlineTipFunction可選，裝置上線時的提示回撥

查詢 Hooks ​

Hook返回值說明
useDevice()DeviceModel | null取得當前裝置資訊
useDeviceProps()PropsModel | null取得 App 傳入的初始參數
useDeviceOnline()boolean取得裝置線上狀態
useDeviceState()number取得裝置狀態碼（位元遮罩）
useDeviceFamily()FamilyModel | null取得家庭資訊
useDeviceArea()string | null取得區域資訊
useDeviceProductUrl()string | null取得產品說明書連結
useDeviceIsOnlyBle()boolean裝置是否為純藍牙裝置
useDeviceIsConnecting()boolean裝置是否正在連線中
useDeviceWsOnline()booleanWebSocket 通道是否線上
useDeviceLanOnline()boolean區域網路通道是否線上
useDeviceBleOnline()boolean藍牙通道是否線上
useEndUserId()string | undefined取得終端使用者 ID
useUserAccount()string | undefined取得當前使用者帳號
useDeviceStoreState()States & Actions取得所有狀態（慎用，易引起頻繁刷新）

操作 Hooks ​

Hook返回值說明
useSetProps()(props) => PropsModel設定初始參數並同步更新裝置
useUpdateDevice()(device) => void更新裝置資訊
useUpdateDeviceName()(name) => void更新裝置名稱
useUpdateOnlineStatus()(state) => void更新線上狀態
useGetDeviceStatus()() => Promise<number>主動拉取裝置狀態
useCheckCurrentDeviceExist()() => Promise<boolean>檢查當前裝置是否仍存在

擴充 ​

  • getDeviceStore()：取得 Store 實例（需先呼叫 createDeviceStore 初始化）。

裝置列表狀態管理 ​

初始化 ​

createDeviceListStore(params?) ​

初始化裝置列表 Store（單例模式）。

參數類型預設值說明
pageSizenumber15每頁裝置數量
isSubScribeAllDevicebooleanfalse是否訂閱所有裝置
processDeviceList(device) => T—自訂裝置列表資料處理函式
subscribeCodeListstring[]—訂閱裝置物模型 code 列表

查詢 Hooks ​

Hook返回值說明
useDeviceList()DeviceModel[]取得裝置列表
useDeviceListStoreState()States & Actions取得裝置列表所有狀態

操作 Hooks ​

Hook返回值說明
useDeviceListRefresh()(success?, fail?) => void重新整理裝置列表（重置為第一頁）
useDeviceListLoadMore()(success?, fail?) => void載入下一頁裝置

擴充 ​

  • getDeviceListStore()：取得 Store 實例（需先呼叫 createDeviceListStore 初始化）。

事件監聽 ​

useInitUpdateDeviceStateEvent(stateCallback?, connectingCallback?) ​

監聽裝置線上/連線狀態變化，通常在入口元件中呼叫一次。

參數說明
stateCallback在離線狀態更新時觸發
connectingCallback連線狀態更新時觸發

useInitDeviceInfoUpdateEvent(callback?) ​

監聽裝置資訊更新事件（如 App 通知裝置已切換/更新）。

useInitDeviceListUpdateEvent(params?) ​

監聽裝置列表更新事件。

參數類型說明
enableAutoCheckoutDeviceboolean是否自動切換裝置（當前裝置被刪除時切換到列表第一個）
refreshDeviceListEventCallback(event) => void列表更新事件回撥
refreshDeviceListSuccessCallback(list) => void列表重新整理成功回撥

注意

以上事件監聽 Hook 應在入口元件中僅呼叫一次，避免重複註冊。

監聽事件名稱 ​

事件常數值說明
EVENT_TYPE_DEVICE_ONLINEonDeviceOnlineState裝置在離線狀態更新
EVENT_TYPE_DEVICE_CONNECTINGonDeviceConnectingState裝置連線狀態更新
EVENT_TYPE_REFRESH_DEVICE_LISTQuecRefreshDeviceList裝置列表更新
EVENT_TYPE_REFRESH_DEVICEQuecRefreshDeviceInfo裝置資訊更新（更換裝置）

裝置列表工具方法 ​

changeDevicePanel(newDevice) ​

向 App 發起面板切換請求。如果新裝置歸屬同一面板，App 僅會發出 QuecRefreshDeviceInfo 事件通知；否則會重新載入新面板。

checkoutEmptyPanel() ​

當裝置列表為空時，主動切換到兜底面板。

回撥管理器 ​

deviceKitParamsManager ​

全域單例，用於註冊面板級別的 UI 回撥。

ts
deviceKitParamsManager.registerCallbacks({
  loadingCallback: () => { /* 顯示 loading */ },
  loadingDismissCallback: () => { /* 隱藏 loading */ },
  tipCallback: (msg) => { /* Toast 提示 */ },
})

裝置線上狀態說明 ​

裝置線上狀態透過位元遮罩表示，支援以下通道組合：

狀態列舉值說明
ALL_OFFLINE0全部離線
WS_ONLINE1WebSocket 線上
WIFI_ONLINE2Wi-Fi 線上
WS_WIFI_ONLINE3WebSocket + Wi-Fi
BLE_ONLINE4藍牙線上
WS_BLE_ONLINE5WebSocket + 藍牙
WIFI_BLE_ONLINE6Wi-Fi + 藍牙
ALL_ONLINE7全部線上

可透過 useDeviceWsOnline()、useDeviceLanOnline()、useDeviceBleOnline() 分別取得各通道狀態。

類型定義 ​

DeviceModel ​

裝置資訊類型，主要欄位：

欄位類型說明
deviceKeystring裝置唯一識別碼
productKeystring產品識別碼
deviceNamestring裝置名稱
onlineStatusnumber線上狀態（0 離線 / 1 線上）
accessTypestring存取類型（0 直連 / 1 閘道 / 2 子裝置）
networkTypestring網路類型（1 WiFi / 2 蜂窩 / 3 NB / 5 藍牙）
deviceTypenumber裝置類型（1 自有 / 2 分享）
capabilitiesBitmasknumber能力位元遮罩

PropsModel ​

App 傳入的初始參數類型，主要欄位：

欄位類型說明
deviceDeviceModel | string裝置資訊
areastring環境區域（如 china）
familyModeEnabledboolean家居模式是否啟用
familyRolestring家庭角色
endUserIdstring終端使用者 ID
userAccountstring當前使用者帳號

FamilyModel ​

家庭資訊類型：

欄位類型說明
familyModeEnabledboolean家居模式是否啟用
familyRolestring家庭角色

裝置配置常數 ​

常數值說明
DEVICE_TYPE_BIND1綁定裝置
DEVICE_TYPE_SHARE2分享裝置
DEVICE_ACCESS_TYPE_DIRECTLY0直連裝置
DEVICE_ACCESS_TYPE_GATEWAY1閘道裝置
