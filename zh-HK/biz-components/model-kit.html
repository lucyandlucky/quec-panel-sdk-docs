ModelKit ​

@quec/panel-model-kit 是一個面向移聯萬物（QuecCloud）物聯網平台的裝置物模型處理庫。它基於 Zustand 狀態管理，提供了從雲端拉取物模型定義、監聽裝置上報、下發控制指令到本地狀態同步的完整閉環能力。

安裝 ​

bash
npm install @quec/panel-model-kit
bash
yarn add @quec/panel-model-kit

注意

本庫依賴以下 peerDependencies，請確保它們已在宿主專案中安裝：@quec/panel-device-kit、@quec/rn-device-module、@quec/rn-log-module、@quec/rn-router-module、bignumber.js、i18n-js、zustand、react、react-native。

快速開始 ​

1. 初始化 Store ​

在應用最頂層（如 App.tsx）建立物模型 Store，並匯出 useModel Hook：

tsx
// App.tsx
import {
  createDeviceModelStore,
  createUseModel,
  BooleanTSLModel,
  NumberTSLModel,
  EnumTSLModel,
} from '@quec/panel-model-kit'

// 定義裝置物模型類型
type MyModelType = {
  powerSwitch: BooleanTSLModel
  temperature: NumberTSLModel
  workMode: EnumTSLModel
}

const store = createDeviceModelStore<MyModelType>()
export const useModel = createUseModel<MyModelType>(store)

TIP

透過泛型 <MyModelType> 傳入你的物模型類型定義，使用 useModel() 時 IDE 會自動補全所有屬性名和類型。

2. 註冊事件監聽並取得物模型 ​

在入口容器元件中完成初始化和事件監聽註冊：

tsx
// AppContainer.tsx
import { useEffect } from 'react'
import {
  useModelInit,
  useInitUpdateDPSEvent,
  modelKitParamsManager,
} from '@quec/panel-model-kit'

const AppContainer = ({ device }) => {
  const initModel = useModelInit()

  useEffect(() => {
    if (device) {
      initModel(device.productKey, device.deviceKey)
    }
  }, [initModel, device])

  // 註冊裝置上報監聽
  useInitUpdateDPSEvent({
    reportTSLCallBack: (event) => { /* 屬性上報回撥 */ },
    eventInfoCallBack: (event) => { /* 事件上報回撥 */ },
    eventLocationCallBack: (event) => { /* 定位資訊回撥 */ },
    eventCallBack: (event) => { /* 其他事件回撥 */ },
  })

  // 註冊全域 UI 回撥
  modelKitParamsManager.registerCallbacks({
    loadingCallback: global.loading,
    loadingDismissCallback: global.loadingDismiss,
    tipCallback: global.toast,
  })

  return <>{/* ... */}</>
}

注意

useInitUpdateDPSEvent 應在入口元件中僅呼叫一次，避免重複註冊導致事件重複處理。

3. 在業務頁面中使用 ​

tsx
// AnyPage.tsx
import { useModel } from './App'
import { useTslWriter, useTslRead } from '@quec/panel-model-kit'

const AnyPage = () => {
  const models = useModel()
  const writeDPS = useTslWriter({ wait: 1000 })
  const readDPS = useTslRead({ wait: 1000 })

  // 下發指令
  const handleTogglePower = (newValue: boolean) => {
    writeDPS({
      data: models.powerSwitch,
      value: newValue,
      success: () => console.log('下發成功'),
      fail: (err) => console.log('下發失敗', err),
    })
  }

  // 主動讀取
  const handleRefresh = () => {
    readDPS({
      data: models.temperature,
      success: () => console.log('讀取成功'),
    })
  }

  return (
    <>
      <Text>溫度：{models.temperature?.attributeValue}</Text>
      <Switch value={models.powerSwitch?.attributeValue} onChange={handleTogglePower} />
    </>
  )
}

API 參考 ​

狀態管理 ​

createDeviceModelStore<T>(params?) ​

初始化全域物模型 Store（單例模式，多次呼叫返回同一實例）。

參數類型說明
modelClassListArray<{ classFilterMethod, listName }>可選，將物模型按 code 規則自動歸類
translateCode(code: string) => string可選，code 轉換函式，預設 camelCase
avoidReadingTSLIdnumber[]可選，初始化時跳過讀取的物模型 ID 列表

createUseModel<T>(store) ​

基於已建立的 Store 產生 useModel Hook，返回物模型集合。

useModelInit() ​

返回初始化函式，呼叫後從雲端拉取物模型並自動讀取一次當前值。

ts
const initModel = useModelInit()
initModel(productKey, deviceKey)

useSetModel() ​

返回手動設定物模型狀態的方法。

getDeviceModelStore() ​

取得 Store 實例，可用於直接訂閱細粒度狀態變化。

INFO

必須在 createDeviceModelStore() 呼叫之後使用，否則會拋出例外。

Store 狀態欄位 ​

欄位類型說明
modelsT物模型集合，整體參考變化
modelsShallowT物模型集合，僅內部單屬性變化，參考不變
modelUpdateTsRecord<string, number>各屬性的最近更新時間戳
isEmptyTslboolean雲端返回的物模型為空
isRequestTslFailedboolean請求物模型介面失敗
rawDpsDPS[]最近一次上報的原始 DPS 資料

事件監聽 ​

useInitUpdateDPSEvent(callbacks) ​

註冊裝置物模型狀態更新的全域 Hook，監聽 Native 層的 onDeviceDpsUpdate 事件。

回撥參數說明
reportTSLCallBack物模型屬性上報時觸發
eventInfoCallBack物模型事件上報時觸發
eventLocationCallBack定位資訊上行時觸發
eventCallBack以上類型均未匹配時的兜底回撥

下發物模型 ​

useTslWriter(options?) ​

返回一個節流包裝後的寫函式，內建裝置線上判斷、BLE/自動通道選擇和 Loading 狀態管理。

配置參數：

參數類型預設值說明
waitnumber1000節流時間間隔（ms）
leadingbooleantrue是否在間隔開始時立即執行
trailingbooleantrue是否在間隔結束時補執行
autoLoadingbooleantrue是否自動管理 Loading

呼叫參數：

參數類型說明
dataAnyTSL | AnyTSL[]要下發的物模型
valueany | any[]對應的屬性值
success() => void成功回撥
fail(err?) => void失敗回撥
offlineFail() => void裝置離線時的回撥
waitResponseboolean是否等待裝置回覆確認

writeFn(params) ​

不含裝置狀態判斷和通道選擇的純寫方法，適用於自訂邏輯場景。

讀取物模型 ​

useTslRead(options?) ​

返回一個節流包裝後的讀函式。

ts
const readDPS = useTslRead({ wait: 1000 })
readDPS({
  data: models.temperature,
  success: () => {},
  fail: (err) => {},
})

useTslAllRead(options?) ​

一次性讀取所有物模型屬性。

回撥管理器 ​

modelKitParamsManager ​

全域單例，用於註冊面板級別的 UI 回撥。

ts
modelKitParamsManager.registerCallbacks({
  loadingCallback: () => { /* 顯示 loading */ },
  loadingDismissCallback: () => { /* 隱藏 loading */ },
  tipCallback: (msg) => { /* Toast 提示 */ },
})

進階用法 ​

監聽單個物模型變化 ​

透過 getDeviceModelStore() 取得 Store 實例，利用 subscribeWithSelector 實現精準的單屬性訂閱：

tsx
import { getDeviceModelStore } from '@quec/panel-model-kit'

const AnyPage = () => {
  useEffect(() => {
    const store = getDeviceModelStore()
    const unsubscribe = store.subscribe(
      state => state.modelsShallow['powerSwitch'],
      (current, previous) => {
        console.log('開關狀態變化：', previous, '→', current)
      },
      { fireImmediately: false }
    )
    return () => unsubscribe()
  }, [])

  return <>{/* ... */}</>
}

TIP

使用 modelsShallow 配合 subscribe 可以避免整個元件因無關屬性更新而重新渲染，顯著提升面板效能。

自訂物模型分類 ​

在初始化 Store 時，透過 modelClassList 參數可將物模型按 code 匹配規則自動歸入陣列分類：

ts
const store = createDeviceModelStore<{
  basicModels: AnyTSL[]
  advancedModels: AnyTSL[]
}>({
  modelClassList: [
    {
      listName: 'basicModels',
      classFilterMethod: (code) => code.startsWith('basic_'),
    },
    {
      listName: 'advancedModels',
      classFilterMethod: (code) => code.startsWith('adv_'),
    },
  ],
})

手動組裝底層方法 ​

如果業務邏輯複雜不適合直接使用封裝的 Hook，可以拆解使用底層方法：

ts
import {
  prepareWriteDps,     // 轉換物模型為 DPS 格式
  prepareReadDps,      // 轉換物模型為讀取 DPS 格式
  ReadDpsWithMode,     // 底層讀方法
  writeFn,             // 底層寫方法
} from '@quec/panel-model-kit'

多語言配置 ​

註冊 tipCallback 回撥後，以下 i18n Key 會在對應操作時自動觸發提示：

Key預設中文觸發場景
quec_model_kit_succeed_set設定成功下發成功
quec_model_kit_failed_set設定失敗下發失敗
quec_model_kit_write_offline裝置離線，請稍後再試裝置不線上時嘗試下發
